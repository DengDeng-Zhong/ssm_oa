报销单处理模块：填写、个人申报以及待处理报销单、修改报销单、提交报销单、审核报销单、打款

填写报销单：就是保存报销单的信息，它包括报销单的基本信息和附属的条目信息，正因为我们提交的信息是比较复杂的，
            所以我们要想收集这些信息，以现有的实体类是没有办法做载体的，这个时候我们就需要考虑声明一个dto
            来进行这个页面与控制器之间的数据传递。对于页面来说，砸门这个报销单条目的数量是不确定的，
            所以又要涉及到不定项表单处理的，   在填写报销单的页面中，包括报销单的基本信息，事由以及总金额。然后它还包括
            条目信息,这个条目信息使用javascript去实现一个动态的多项的不定项的条目数的一个设置.填写完进行保存,保存之后,
            又要转向的某一个位置去,当然合理的转向是转向到一个详情列表.这个列表包括报销单的基本信息以及对应的条目信息还有这个
            报销单审核流程相关的信息.
            刚刚也说了,收集的数据是比较特殊的,需要声明一个dto,在里面去声明需要收集的数据,报销单对象的基本信息和一系列条目信息

个人报销单、待处理报销单：处理功能按钮显示（一个页面，两个控制器方法，一个业务方法）
修改报销单：1.无非就是修改基本信息和条目信息变动的一些更新，条目信息变动是相对复杂的，他有可能要先插入一些条目，原有的一些条目可能会删掉，
            原有的条目可能只是部分内容发生一些变动
            2.修改报销单需要传递报销单的编号，我使用的方法是使用隐藏域的方式去传递
提交报销单：他的实现分为两步,第一是修改状态,第二是处理流程的记录,到这个时候功能就进入了处理流程,每一步的操作都须要记录,我们需要考虑的是部门经理的获取
            部门经理的获取其实是给报销单设置待处理人,待处理人包括部门经理,财务,总经理
审核报销单、打款：这是两个独立的功能，只不过他们的处理流程和界面是一样的，所有是合在一起实现的，他的实现流程是修改状态和记录处理流程，
                  在实现过程中需要考虑的是进行不同的操作，那么对报销单的影响以及对审核记录的保存产生不一样的效果，审核如果通过的话，还要考虑是否需要复审这种情况。
